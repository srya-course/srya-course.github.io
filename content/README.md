# 網站內容管理指南

歡迎！此資料夾用於管理部落格的所有內容。所有內容都使用 YAML 格式進行管理。

## 目錄結構

```
content/
├── diary/        # 日記管理
├── notes/        # 筆記管理
├── gallery/      # 圖片管理
├── videos/       # 影片管理
└── README.md     # 本檔案
```

## 快速開始

### 1. 新增日記

1. 進入 `diary/` 資料夾
2. 複製 `example.yaml` 為新檔案，例如 `2024-12-04-my-diary.yaml`
3. 按照範本編輯內容
4. 保存後，網站會自動顯示

📖 詳細說明請見 `diary/README.md`

### 2. 新增筆記

1. 進入 `notes/` 資料夾
2. 複製 `example.yaml` 為新檔案，例如 `2024-12-04-react-notes.yaml`
3. 按照範本編輯內容
4. 保存後，網站會自動顯示

📝 詳細說明請見 `notes/README.md`

### 3. 新增圖片

1. 進入 `gallery/` 資料夾
2. 將圖片檔案放在 `images/` 子資料夾中
3. 編輯 `gallery.yaml`，在 `images` 陣列中添加新的圖片條目
4. 保存後，網站會自動顯示

🖼️ 詳細說明請見 `gallery/README.md`

### 4. 新增影片

1. 進入 `videos/` 資料夾
2. 將縮圖檔案放在 `thumbnails/` 子資料夾中（可選）
3. 編輯 `videos.yaml`，在 `videos` 陣列中添加新的影片條目
4. 保存後，網站會自動顯示

🎬 詳細說明請見 `videos/README.md`

## 內容管理方式

### 日記和筆記（個別 YAML 檔案）

- **日記**：每篇日記一個 YAML 檔案
- **筆記**：每篇筆記一個 YAML 檔案
- 檔案名稱可自訂，但副檔名必須是 `.yaml`
- 避免使用 `example.yaml`（這是範本檔案）

### 圖片和影片（統一 YAML 檔案）

- **圖片**：所有圖片信息統一在 `gallery/gallery.yaml` 中管理
- **影片**：所有影片信息統一在 `videos/videos.yaml` 中管理

## 重要提示

### ✅ 應該做的事

- ✓ 遵循 YAML 格式規則
- ✓ 填寫所有必需欄位
- ✓ 確保 `id` 的唯一性
- ✓ 使用 `YYYY-MM-DD` 格式的日期
- ✓ 定期備份內容檔案

### ❌ 不應該做的事

- ✗ 修改網站上的內容後期望永久保存（使用本資料夾管理）
- ✗ 手動修改編譯後的檔案
- ✗ 刪除 `example.yaml`（它是重要的範本）

## YAML 格式基礎

YAML 是一種人類友善的數據格式。以下是一些基本規則：

```yaml
# 這是註解
key: value          # 鍵值對
string: "文字"      # 字符串
number: 42          # 數字
boolean: true       # 布爾值
list:               # 列表
  - item1
  - item2
multiline: |        # 多行文本
  這是第一行
  這是第二行
```

## 支援的日期格式

- ISO 8601：`2024-12-04`（推薦）
- 其他格式可能也支援，但推薦使用 `YYYY-MM-DD`

## 字符編碼

所有文件應使用 UTF-8 編碼以正確顯示中文和其他字符。

## 疑難排解

### 內容未顯示

1. 檢查 YAML 語法是否正確（可使用線上 YAML 驗證器）
2. 確認所有必需欄位都已填寫
3. 檢查檔案副檔名是否為 `.yaml`
4. 檢查 `id` 是否唯一

### 日期顯示錯誤

1. 確認日期格式為 `YYYY-MM-DD`
2. 檢查日期是否有效（例如，2024-02-30 無效）

### YAML 語法錯誤

常見錯誤：
- 混合使用空格和制表符縮進（YAML 只接受空格）
- 冒號後面必須有空格（`key: value` 正確，`key:value` 錯誤）
- 字符串如果包含特殊字符需要用引號括起來
